## Missed Gaps

- **G1 — `zerops_recipe` init surface is ungoverned and incomplete.** Surface: `internal/recipe/handlers.go:164` emits the MCP description, but it omits supported `record-fragment`; the schema and dispatcher include it (`internal/recipe/handlers.go:107`, `internal/recipe/handlers.go:273`). Drift mechanism: governance misses this package because description drift scans `../tools/*.go` only (`internal/tools/description_drift_test.go:48`, `internal/tools/description_drift_test.go:64`) and annotation tests omit `zerops_recipe` (`internal/tools/annotations_test.go:69`, `internal/tools/annotations_test.go:88`). Repro: `recipe-active` v3, `zerops_recipe action=status` returns current phase guidance, then `record-fragment` is required by fragments, while MCP init description never names it.
- **G2 — v3 recipe sessions are invisible to MCP init and `zerops_workflow status`.** Surface: init hint is built before/without `recipeStore` (`internal/server/server.go:81`, `internal/server/server.go:83`, `internal/server/server.go:141`, `internal/server/server.go:188`); `ComposeStateHint` sees only v2 workflow registry and work sessions (`internal/server/instructions.go:72`, `internal/server/instructions.go:92`). Drift mechanism: v3 store is in-memory and separate (`internal/recipe/handlers.go:17`, `internal/recipe/handlers.go:28`), while workflow status routes only v2 recipe/bootstrap or lifecycle (`internal/tools/workflow.go:234`, `internal/tools/workflow.go:242`). Repro: `recipe-active` v3 after compaction, agent calls canonical-but-wrong `zerops_workflow action=status`; response can be idle/develop lifecycle, not `zerops_recipe action=status`.
- **G3 — corrupt work-session file defeats compaction recovery.** Surface: bad JSON returns `unmarshal work session` (`internal/workflow/work_session.go:122`, `internal/workflow/work_session.go:124`); `ComputeEnvelope` propagates it (`internal/workflow/compute_envelope.go:61`, `internal/workflow/compute_envelope.go:70`); `status` returns a leaf error with empty suggestion (`internal/tools/workflow.go:759`, `internal/tools/workflow.go:762`, `internal/tools/workflow.go:765`). Drift mechanism: the recovery primitive depends on the file being parseable, while init silently drops the same error (`internal/server/instructions.go:92`). Repro: `develop-active` or `develop-closed-auto`, corrupt `.zcp/state/work/{pid}.json`, then call `zerops_workflow action=status` after compaction.
- **G4 — `idle` + orphan incomplete meta lacks a service snapshot, so resume atoms cannot fire.** Surface: `IdleIncomplete` is derived only from `ServiceSnapshot.Resumable` (`internal/workflow/compute_envelope.go:125`, `internal/workflow/compute_envelope.go:140`), but snapshots are built only by iterating live platform services (`internal/workflow/compute_envelope.go:176`, `internal/workflow/compute_envelope.go:184`); `Resumable` is set only inside a live-service snapshot (`internal/workflow/compute_envelope.go:216`, `internal/workflow/compute_envelope.go:221`). Drift mechanism: state truth in ServiceMeta disappears when platform state disagrees. Coverage fixtures include idle empty/bootstrapped/adopt but no idle-incomplete case (`internal/workflow/corpus_coverage_test.go:42`, `internal/workflow/corpus_coverage_test.go:90`). Repro: `bootstrap-active` interrupted after partial meta, live service deleted externally, then `zerops_workflow action=status` in idle: envelope goes empty/adoptable path, not resume/cleanup.
