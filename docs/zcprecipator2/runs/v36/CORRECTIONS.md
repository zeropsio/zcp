## 2. Why the first pass failed (meta-analysis)

Three structural causes, in order of severity:

### Cause A — Artifact-shape-as-proxy-for-depth

HANDOFF-to-I7 said "produce 5 artifacts in this shape". I produced 5 artifacts in the right shape. The shape was convincing enough that nothing in the process forced verification of the claims inside. The `README.md` defect-closure table read as thorough. The `analysis.md` per-defect verification sections read as evidence-based. The `verdict.md` read as considered. None of it required me to have actually Read the on-disk files being graded.

Shipping the artifact-shape felt like done. It wasn't.

### Cause B — "Unmeasured — phase unreached" as universal alibi

The v35 calibration-bars sheet I carried forward had entire sections labeled as unmeasurable because v35 stalled. I adapted those labels to v36: "content quality cold-read pending close step", "editorial-review bars unmeasurable — not dispatched", "gotcha-origin ratio requires manual classification".

Every one of these was an alibi. The content files were on disk. I could have Read every writer-authored file end-to-end and graded them. I didn't. I inherited v35's "unmeasurable" framing and applied it to v36 unchanged, without asking "is this still unmeasurable now that the files exist?"

### Cause C — No evidence-to-claim binding

My verdict said "F-1 PASS, envelope fires at readmes with 15 dispatch-brief-atom calls, max 11.5 KB". That sounds authoritative. It was a paraphrase of what the flow extractor summary told me. I never grepped the raw JSONL to verify. If the flow extractor had miscounted, my verdict would have been wrong and nobody could audit it — the claim wasn't tied to a specific `main-session.jsonl` row or `dispatch-brief-atom` response body.

## 3. What would have caught these defects

A mechanical analysis harness that runs **before** the analyst writes prose. Specifically:

- **Structural integrity sweep**: `find environments/ -type d` compared against canonical set → catches F-9 mechanically.
- **Tarball content inventory**: `tar -tzf <archive> | grep README.md` → catches F-10 mechanically.
- **Marker exact-form regex**: grep with anchored regex → catches F-12 mechanically.
- **File-presence inventory**: compare writer's reported output list to on-disk files → catches F-10 + F-13 mechanically.
- **Retry-round counter**: parse `main-session.jsonl` for every `checkResult.passed==false` → catches F-14/15/16 mechanically.

None of these measurements require analyst judgment. All are `grep` / `find` / `jq` one-liners. The harness design is at [`../spec-recipe-analysis-harness.md`](../spec-recipe-analysis-harness.md). Phase 1 of [`../HANDOFF-to-I8-v37-prep.md`](../HANDOFF-to-I8-v37-prep.md) commissions its build.

## 6. Process changes adopted for v37

From [`../HANDOFF-to-I8-v37-prep.md`](../HANDOFF-to-I8-v37-prep.md) §6 (Analysis discipline rules):

1. No verdict without machine-report SHA in front matter.
2. No PASS/FAIL claim without evidence citation within 50 chars.
3. No "unmeasured" cells in the checklist when files are on disk.
4. Read every writer-authored file before grading it (Read-receipt enforced).
5. Diff every dispatch prompt against Go source.
6. Retry-cycle attribution required for every failing check round.
7. No verdict ship before checklist is 100% filled.
8. Tripwire on self-congratulatory language without citation.

These rules are enforced by pre-commit hooks, not just exhortation. The commit hook refuses verdicts failing the rules.

## 7. If you're reading this as a fresh analysis instance

You are likely reading this after `HANDOFF-to-I8-v37-prep.md` directs you here. The corrections above are the context for why the v37 handoff demands a harness before analysis. Do not repeat my v36 failure:

- **Don't ship on artifact-shape.** The harness checks structural bars; the checklist requires read-receipts; the commit hook enforces citation. Trust the mechanical layer.
- **Don't use "unmeasured" as an alibi.** If the file is on disk, measure it.
- **Don't paraphrase evidence.** Cite the raw source (`flow-main.md row N` or `main-session.jsonl line M` or `<file>:<line>`).

If you catch yourself about to ship a verdict that feels convincing but can't survive a 30-second challenge from the user pointing at evidence, stop. That's the exact v36 failure mode.
