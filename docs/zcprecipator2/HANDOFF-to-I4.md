## C-7e design decisions — locked in by I3 (my recommendations)

These are my proposed resolutions. If you disagree, raise before coding — but they have internal consistency and match the spec's "thin CLI adapter" directive.

1. **Output format: plain text by default, `--json` opt-in.**
   Default stdout line per check row: `PASS <name>` or `FAIL <name>: <detail>`. Exit 0 if all rows pass; exit 1 if any fail. Machine-parseable JSON via `--json` flag: one JSON object per check row on its own line (ndjson) for easy streaming / jq filtering.

2. **Per-shim flag convention matches `check-rewrite.md §18`.**
   `zcp check env-refs --hostname=X --path=./` — hostname identifies the target codebase; path is the mount root the shim walks. Other shims per the §18 list:
   - `--hostname=X` required by: env-refs, run-start-build-contract, env-self-shadow, ig-code-adjustment, ig-per-item-code, comment-specificity, yml-schema, kb-authenticity, worker-queue-group-gotcha, worker-shutdown-gotcha.
   - `--mount-root=./` required by: manifest-honesty, manifest-completeness, cross-readme-dedup, symbol-contract-env-consistency.
   - `--env=N` required by: comment-depth, factual-claims.
   - Extra: `manifest-completeness --facts=/tmp/zcp-facts-{sess}.jsonl` (facts-log path).

3. **State / plan sourcing for shims that need them: `--state=<dir>` with graceful fallback.**
   `checkSymbolContractEnvVarConsistency` needs `plan.SymbolContract.EnvVarsByKind`. `checkManifestCompleteness` needs the facts-log path. The shim reads `.zcp/state/recipe-plan.json` (or whatever the engine serializes) when `--state=<dir>` is passed; when absent, the shim runs in **degraded mode** — degraded is "vacuous pass" per the existing predicate contract (`CheckSymbolContractEnvVarConsistency` already short-circuits on empty contract per `ops/checks/symbol_contract_env.go`).

4. **`yml-schema`: `--schema-json=<path>` or graceful no-op.**
   `CheckZeropsYmlFields(ctx, ymlDir, validFields *schema.ValidFields)` returns nil when `validFields == nil`. The shim accepts `--schema-json=<path>` to hydrate validFields; when absent, the shim prints a one-line skip note on stderr and exits 0. No network fetch (avoids making shims flaky on agent containers without internet).

5. **Integration test: one test function, table-driven, one row per subcommand.**
   Fixture: a single tempdir with a minimal-but-complete recipe mount (one `apidev/` with `zerops.yaml` + `README.md` + `.env.example` + `ZCP_CONTENT_MANIFEST.json`; one `environments/local/import.yaml`). Each table row invokes one subcommand against the fixture, asserts the exit code + a substring of stdout. Not exhaustive — the ops/checks package owns the predicate-behavior tests. This test proves the CLI plumbing works + each subcommand reaches its predicate.

6. **Subagent-dispatch pattern: NOT needed for C-7e.**
   The 16 shim files are structurally similar but each touches a shared parent registry in `check.go`. Keeping authorship in the main instance avoids the cross-file conflict concern that made subagent-dispatch a worse fit than vertical sequential authoring (the same reasoning that led to splitting C-7 into 5 sub-commits). Expected wall: ~2-3 hours single-thread.
